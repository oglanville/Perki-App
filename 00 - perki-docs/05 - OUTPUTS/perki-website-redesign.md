## 1. SUMMARY OF THE UI UX PRO MAX DESIGN SYSTEM

**Category:** Luxury/Premium Brand · **Recommended style:** Liquid Glass · **Page pattern:** Minimal Single Column.

**Colour palette (exact):**

| Role | Hex | Meaning |
|------|-----|---------|
| Primary | `#F59E0B` | Gold — trust |
| Secondary | `#FBBF24` | Light gold — highlight |
| CTA / Accent | `#8B5CF6` | Purple — tech |
| Background | `#0F172A` | Slate-900 — premium dark |
| Text | `#F8FAFC` | Near-white |

System note: *"Gold trust + purple tech."*

**Typography:** IBM Plex Sans for both heading and body (weights 300–500–600–700). Mood: financial, trustworthy, professional, banking. Single-family pairing → hierarchy comes from weight + size, not contrasting families.

**Spacing scale:** `4 · 8 · 16 · 24 · 32 · 48 · 64` (xs→3xl). 16px = standard padding, 24px = section padding, 48px = section margins, 64px = hero padding.

**Shadow depths:** sm `0 1px 2px/.05` → md `0 4px 6px/.1` (cards/buttons) → lg `0 10px 15px/.1` (modals) → xl `0 20px 25px/.15` (hero/featured).

**Component rules:** Primary button = purple `#8B5CF6`, 12×24px, radius 8, weight 600, 200ms, hover `opacity .9 + translateY(-1px)`. Secondary = transparent + 2px gold border. Cards = dark, radius 12, 24px padding, shadow-md, hover lift `translateY(-2px)` to shadow-lg. Inputs = 16px font (no iOS zoom), focus = gold border + 3px gold glow.

**Layout pattern (Minimal Single Column):** single CTA focus, large type, generous whitespace, no nav clutter, mobile-first. Section order: Hero headline → short description → max 3 benefit bullets → CTA → footer. Landing supplement (matched): **Hero + Testimonials + CTA** — social proof *before* the CTA, 3–5 testimonials with photo+name+role, CTA sticky in hero and repeated after proof.

**Style — Liquid Glass:** flowing translucent glass, morphing, dynamic backdrop-blur, fluid 400–600ms easing, iridescent accents. Best for premium/luxury brands.

**Anti-patterns (forbidden):** cheap visuals · fast animations · emojis as icons (use SVG: Heroicons/Lucide) · missing `cursor:pointer` · layout-shifting hover scales · text below 4.5:1 contrast · instant state changes · invisible focus states.

**Touch/UX (CRITICAL):** 44×44px min targets · ≥8px gap between targets · tap (not hover) for primary actions · `overscroll-behavior: contain` to kill accidental pull-to-refresh · responsive at 375 / 768 / 1024 / 1440.

## 2. PERKI REDESIGN STRATEGY

**The one-line truth:** *Perki shows you the money you're already paying for — and aren't using.*

**Strategic pillars (each maps to a system rule):**

1. **Lead with the loss, not the feature.** The hero quantifies wasted value ("The average UK household pays for £200+ of perks a month. Most go unclaimed."). Minimal Single Column → one headline, one promise, one CTA. No feature-soup above the fold.
2. **Trust is the product.** It's fintech touching people's subscriptions. Gold (`#F59E0B`) carries trust signals (FSCS-style reassurance, provider logos, "we never touch your money"); purple (`#8B5CF6`) carries every action. Colour does the heavy lifting the system intends.
3. **Show real perks, instantly.** Replace abstract marketing with a live, scrollable preview of actual perks (Monzo, Revolut, Sky, OVO) rendered in glass cards. Proof over claims.
4. **Mobile is the product surface.** Designed at 375px first; everything thumb-reachable; sticky CTA always one tap away. Desktop is the *enhancement*, not the source.
5. **Premium restraint.** Liquid Glass used as *accent* (hero orb, card surfaces, sticky bar) — never wall-to-wall, honouring the "no cheap visuals / no fast animations" anti-patterns and the contrast floor.

**Primary conversion goal:** email/app-signup. **Secondary:** "See your perks" interactive preview. **North-star metric:** signups → first perk claimed.

**Positioning vs. competitors:** Monzo/Revolut/Amex surface only *their own* perks; TopCashback/Honey are checkout coupons. Perki is the only **cross-provider personal perk inventory**. The site says this in one breath and proves it with the multi-brand preview.

## 5. COMPONENT LIBRARY

All values from MASTER.md component specs; Perki applies them.

**Button — Primary (purple CTA)**
- `background:var(--perki-purple); color:#fff; padding:12px 24px; border-radius:8px; font-weight:600; min-height:44px; cursor:pointer; transition:all var(--t-micro) var(--ease-fluid);`
- Hover: `opacity:.9; transform:translateY(-1px)`. Active: `translateY(0)`. Focus: `outline:none; box-shadow:0 0 0 3px rgba(139,92,246,.5)`.
- Loading: spinner + `disabled`, label → "One sec…".

**Button — Secondary (gold outline)**
- `background:transparent; color:var(--perki-gold); border:2px solid var(--perki-gold); padding:12px 24px; radius:8px; weight:600; min-height:44px`.

**Perk Card (glass) — signature component**
- `background:var(--perki-glass); backdrop-filter:blur(14px); border:1px solid var(--perki-glass-brd); border-radius:12px; padding:24px; box-shadow:var(--shadow-md); cursor:pointer; transition:box-shadow,transform var(--t-micro)`.
- Hover: `box-shadow:var(--shadow-lg); transform:translateY(-2px)` (no scale → no layout shift).
- Anatomy: provider logo (SVG, 24×24) · tier pill (gold) · perk title (H3) · one-line value · reset chip ("Resets monthly") · chevron.

**Tier Pill:** `--r-pill`, gold-light bg at 14%, gold text, 14px/500, padding 2×10. Tiers: Free / Extra / Perks / Max (Monzo), Standard→Ultra (Revolut), etc.

**Provider Group Header:** collapsible; logo + name + perk count chip; chevron rotates 180° over 200ms.

**Input/Field:** `padding:12px 16px; font-size:16px; radius:8px; border:1px solid var(--perki-glass-brd); background:var(--perki-ink-2)`. Focus: gold border + 3px gold glow. Label always visible (`<label for>`), never placeholder-only.

**Sticky CTA Bar (mobile):** glass bar pinned bottom, safe-area inset, `--z-sticky`; full-width purple button; appears after hero scroll, hides on keyboard open.

**Bottom Tab Bar (app):** 4 items, SVG icons (Lucide), 44px targets, 8px gaps, active = gold icon + label, inactive = muted.

**Modal:** overlay `rgba(0,0,0,.5)` + `backdrop-filter:blur(4px)`; sheet radius 16, shadow-xl, max-width 500 / 90% width; on mobile slides up as bottom sheet.

**Toast:** glass, top inset, auto-dismiss 4s, never colour-only (icon + text).

**Icon system:** Lucide SVG, 24×24 viewBox, `w-6 h-6`. Brand logos from Simple Icons (verified paths). **No emoji icons anywhere** (anti-pattern).

## 8. DEVELOPER-READY SPEC

**Stack:** Existing React 18 + Vite + Supabase (keep). Styling: Tailwind mapped to the tokens in §4 (system default = html-tailwind). Icons: `lucide-react`. Fonts: IBM Plex Sans via Google Fonts `@import`.

**Tailwind theme extension (drop-in):**
```js
theme:{ extend:{
  colors:{ gold:'#F59E0B', goldlight:'#FBBF24', purple:'#8B5CF6',
           ink:'#0F172A', ink2:'#1A2433', snow:'#F8FAFC', muted:'#94A3B8' },
  fontFamily:{ sans:['"IBM Plex Sans"','system-ui','sans-serif'] },
  borderRadius:{ btn:'8px', card:'12px', modal:'16px' },
  boxShadow:{ sm:'0 1px 2px rgba(0,0,0,.05)', md:'0 4px 6px rgba(0,0,0,.1)',
              lg:'0 10px 15px rgba(0,0,0,.1)', xl:'0 20px 25px rgba(0,0,0,.15)' },
  transitionTimingFunction:{ fluid:'cubic-bezier(.4,0,.2,1)' },
}}
```

**Component → Tailwind contracts:**
- Primary btn: `bg-purple text-white font-semibold px-6 py-3 rounded-btn min-h-[44px] cursor-pointer transition-all duration-200 ease-fluid hover:opacity-90 hover:-translate-y-px focus:outline-none focus:ring-[3px] focus:ring-purple/50`
- Glass perk card: `bg-snow/[0.06] backdrop-blur-[14px] border border-snow/[0.12] rounded-card p-6 shadow-md cursor-pointer transition-[box-shadow,transform] duration-200 hover:shadow-lg hover:-translate-y-0.5`
- Input: `px-4 py-3 text-base rounded-btn bg-ink2 border border-snow/[0.12] focus:border-gold focus:ring-[3px] focus:ring-gold/20 focus:outline-none`
- Tier pill: `rounded-full bg-goldlight/[0.14] text-gold text-sm font-medium px-2.5 py-0.5`

**Responsive rules (mobile → tablet → desktop):**

| Breakpoint | Layout |
|------------|--------|
| **375px (base, iPhone 13)** | Single column. Full-width CTAs. Sticky bottom CTA/tab bar. Horizontal-swipe perk previews. Type: Display 40. Container padding 16px. |
| **768px (md)** | 2-col perk grid. Hero centred, max-w-2xl. Display 56. Nav reveals inline links (still minimal). Container padding 24px. |
| **1024px (lg)** | 3-col perk grid. Two-column hero (copy + glass visual). Sticky in-hero CTA. Max-w-6xl, centred. |
| **1440px (xl)** | Same as lg, max-w-7xl, more whitespace (space-3xl section rhythm). Featured cards use shadow-xl. |

**Motion:** micro-interactions 200ms; hero/morphing 480ms `ease-fluid`; **all wrapped in** `@media (prefers-reduced-motion: reduce){ *{animation:none!important; transition:none!important} }`. No animation under 150ms (anti-pattern: "fast animations").

**Accessibility acceptance criteria:** body contrast ≥4.5:1 (snow-on-ink passes); every icon-only button has `aria-label`; visible focus rings; tab order matches DOM; all inputs have `<label for>`; images have alt; colour never sole signal (icons + text on status).

**Performance:** WebP + `srcset` + lazy-load for any imagery; limit `backdrop-filter` to ≤3 simultaneous glass surfaces per viewport (the system's Liquid Glass perf caveat); reserve space for async perk lists to prevent layout jumping; `content-visibility:auto` on long provider groups.

**Z-index scale:** nav 20 · sticky 30 · modal 50 (no ad-hoc values).
